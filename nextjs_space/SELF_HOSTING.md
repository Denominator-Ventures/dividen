# Self-Hosting DiviDen

Run your own DiviDen instance. Full control over your data, your agents, and your federation connections.

---

## Architecture

```
┌─────────────────────────────────────────────────────┐
│                    DiviDen Instance                   │
│                                                       │
│   Next.js 14 (App Router)                             │
│   ├── 12 pages (dashboard, settings, docs, admin)    │
│   ├── 153 API routes                                  │
│   ├── MCP server (/api/mcp) — 20+ tools              │
│   ├── A2A protocol (/.well-known/agent-card.json)    │
│   └── Federation endpoints (/api/federation/*)        │

                    │
                    ▼
            ┌───────────────┐
            │  PostgreSQL    │
            │  (local or     │
            │   hosted)      │
            └───────────────┘
```

## Quick Start (5 minutes)

### Prerequisites

| Requirement | Version | Notes |
|---|---|---|
| Node.js | 18+ (22 recommended) | [nodejs.org](https://nodejs.org) |
| PostgreSQL | 13+ | Via Docker (included) or any hosted provider |
| Docker | Optional | Only needed if using the included `docker-compose.yml` |

### 1. Clone and setup

```bash
git clone https://github.com/Denominator-Ventures/dividen.git
cd dividen/nextjs_space
bash scripts/setup.sh
```

The setup script handles everything: dependencies, `.env` generation, database, migrations, and seed data.

### 2. Start the server

```bash
yarn dev
# or: npm run dev
```

Open **http://localhost:3000**. Log in with the seeded admin account:
- Email: `admin@dividen.ai`
- Password: `DiviDen2024!`

### 3. Create your real account

Visit **http://localhost:3000/setup** and create your own account. This becomes the primary operator of your DiviDen instance.

## Environment Variables

Copy `.env.example` to `.env`. Here's every variable:

### Required

| Variable | Description | Example |
|---|---|---|
| `DATABASE_URL` | PostgreSQL connection string | `postgresql://dividen:dividen@localhost:5432/dividen` |
| `NEXTAUTH_SECRET` | Session encryption key (auto-generated by setup script) | `base64-encoded-random-string` |
| `ADMIN_PASSWORD` | Password for the `/admin` dashboard | `your-secure-password` |

### Optional — AI

| Variable | Description | Default |
|---|---|---|
| `ABACUSAI_API_KEY` | Abacus.AI LLM API key (enables Divi AI features) | — |

> Users can also add their own OpenAI or Anthropic API keys in **Settings → API Keys**.

### Optional — Payments (Stripe)

| Variable | Description | Default |
|---|---|---|
| `STRIPE_SECRET_KEY` | Stripe secret key for marketplace payments | — |
| `STRIPE_PUBLISHABLE_KEY` | Stripe publishable key (client-side) | — |
| `STRIPE_WEBHOOK_SECRET` | Stripe webhook signing secret | — |

### Optional — Fees

| Variable | Description | Default |
|---|---|---|
| `MARKETPLACE_FEE_PERCENT` | Platform fee on marketplace agent executions | `3` (set to `0` for self-hosted) |

### Optional — Email

## Database

### Using Docker (easiest)

```bash
docker compose up -d
```

This starts PostgreSQL 16 on port 5432 with:
- User: `dividen`
- Password: `dividen`
- Database: `dividen`

The default `DATABASE_URL` in `.env.example` already points here.

### Using a hosted database

Set `DATABASE_URL` in `.env` to your connection string. Works with:
- [Neon](https://neon.tech)
- [Supabase](https://supabase.com)
- [Railway](https://railway.app)
- [PlanetScale](https://planetscale.com) (MySQL — not recommended, schema uses PostgreSQL features)
- Any PostgreSQL 13+ instance

### Migrations

```bash
# Apply all migrations
yarn prisma migrate deploy

# Generate Prisma client after schema changes
yarn prisma generate

# View current migration status
yarn prisma migrate status
```

> **Never** run `yarn prisma db push --accept-data-loss` on a database with real data.

### Seed data

```bash
yarn tsx scripts/seed.ts
```

Creates the default admin account and notification rules.

## Production Deployment

### Build

```bash
yarn build
yarn start
```

The app builds as a standalone Next.js server. No external runtime dependencies beyond Node.js and PostgreSQL.

### Environment

Set these in production:

```bash
NODE_ENV=production
NEXTAUTH_URL=https://your-domain.com   # Your public URL
NEXTAUTH_SECRET=<generated-secret>
DATABASE_URL=<production-postgres-url>
ADMIN_PASSWORD=<strong-password>
```

### Reverse proxy (nginx)

```nginx
server {
    listen 443 ssl;
    server_name your-domain.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

### Docker (full stack)

A production Dockerfile is planned but not yet included. For now, the recommended approach is:

1. Build locally or in CI: `yarn build`
2. Copy the `.next/standalone` directory to your server
3. Run with `node .next/standalone/server.js`

## Federation

Federation allows DiviDen instances to communicate with each other — sharing relays, patterns, jobs, and intelligence.

### Enabling federation

1. Go to **Settings → Federation** in your dashboard
2. Set your instance's public URL
3. Generate a federation token

### Connecting instances

1. Share your instance URL with the other operator
2. Create a connection with `isFederated: true` and the peer's instance URL
3. Exchange federation tokens

Federated endpoints:
- `/api/federation/relay` — Cross-instance relay messages
- `/api/federation/patterns` — Ambient pattern exchange (automated every 6h)
- `/api/federation/jobs` — Job gossip (share open positions)
- `/api/federation/graph` — Serendipity matching
- `/api/federation/routing` — Intelligent task routing
- `/api/federation/mcp` — Cross-instance MCP tool invocation
- `/api/federation/reputation` — Portable reputation attestations

## MCP Server

DiviDen exposes an MCP (Model Context Protocol) server at `/api/mcp`.

### Connecting from Claude Desktop

Add to your Claude Desktop config (`claude_desktop_config.json`):

```json
{
  "mcpServers": {
    "dividen": {
      "command": "curl",
      "args": ["-X", "POST", "https://your-domain.com/api/mcp"],
      "env": {
        "AUTHORIZATION": "Bearer YOUR_API_KEY"
      }
    }
  }
}
```

### Available tools

20 static tools + dynamic marketplace agent tools. See `/api/mcp` with `{"method": "tools/list"}` for the full list.

## Self-Hosted Fee Configuration

When self-hosting, you probably want to disable platform fees:

```bash
# In .env
MARKETPLACE_FEE_PERCENT=0    # No marketplace fee
RECRUITING_FEE_PERCENT=0     # No recruiting fee
```

With fees at 0%, all marketplace and recruiting revenue goes directly to the operator/developer.

## Troubleshooting

### Health check

```bash
curl http://localhost:3000/api/status
```

Returns database connection status, user count, and migration state.

### Common issues

| Issue | Fix |
|---|---|
| `@prisma/client did not initialize` | Run `yarn prisma generate` |
| Database connection refused | Check `DATABASE_URL` in `.env`, ensure Postgres is running |
| Migrations failed | Run `yarn prisma migrate status` to see what's pending |
| NEXTAUTH_SECRET mismatch | If you changed the secret, all existing sessions are invalidated — users need to re-login |
| Port 3000 in use | Set `PORT=3001` in `.env` or use `PORT=3001 yarn dev` |

### Logs

- **Application logs**: stdout/stderr from the Node.js process
- **Database query telemetry**: Stored in the `TelemetryEvent` table, visible at `/admin`
- **Rate limit hits**: Returned as `429 Too Many Requests` with `Retry-After` header
