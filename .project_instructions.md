# DiviDen Command Center - Project Instructions

## Overview
DiviDen is an agentic working protocol command center built with Next.js 14, Prisma, and Tailwind CSS. It provides AI agent coordination, contact management, document handling, calendar integration, and inter-agent communication.

## Design System
- Dark theme with #4F7CFF brand blue, #6B9AFF brand secondary
- Space Grotesk headings, JetBrains Mono for labels/mono text
- Background: #0a0a0a base, #111111 surface
- Text: #f5f5f5 primary, #a1a1a1 secondary, #666666 muted
- Borders: rgba(255,255,255,0.06)

## Key Architecture
- Auth: NextAuth with credentials provider, admin password from ADMIN_PASSWORD env var
- Database: Prisma with PostgreSQL
- AI: Abacus.AI LLM API (ABACUSAI_API_KEY)
- Email: IMAP/SMTP integration + Abacus notification API
- Federation: Cross-instance agent communication support

## Admin Dashboard
- `/admin` route with stats, user management, system overview
- Protected by admin check (user email matches admin)

## Proactive Relay Intelligence (Agentic Communication Protocol)
- **Layer 17 system prompt** completely rewritten to instruct Divi on autonomous relay behavior
- **Three relay modes**:
  - `relay_request` — Direct ask to a specific connection's agent
  - `relay_broadcast` — Ask ALL connections at once (team-wide / company-wide)
  - `relay_ambient` — Low-priority ambient ask; receiving agent weaves it into conversation naturally without interrupting
- **Ambient protocol**: The core differentiator — receiving agents don't tell their user "someone wants to know X." They wait for the topic to come up naturally, ask about it as if they're curious, then feed the answer back via `relay_respond`
- **Response surfacing**: System prompt now includes recently completed relay responses (last 24h) with instructions to surface them naturally ("By the way, [name] got back to us about...")
- **Proactive intent detection**: Divi recognizes patterns like "I wonder what [name] thinks" → ambient relay, "ask everyone" → broadcast, "ask [name]" → direct relay
- **Smart routing**: Divi checks connection profiles (skills, task types, capacity) before choosing who to relay to
- **Chief of Staff mode**: More autonomy to proactively send ambient relays without asking first

## Outbound Invitations
- **Prisma model**: `Invitation` with token (uuid), inviter/invitee info, message, status, instanceUrl, sourceInstance, expiresAt (14 days)
- **Email**: Notification type "Connection Invitation" (NOTIF_ID_CONNECTION_INVITATION), styled HTML from `invites@{hostname}`
- **API routes**:
  - `POST /api/invites` — create + send email (or auto-create connection if user exists)
  - `GET /api/invites` — list sent invites
  - `GET /api/invites/[token]` — public peek for signup context
  - `POST /api/invites/[token]/accept` — accept specific invite
  - `POST /api/invites/accept-pending` — auto-accept all pending invites for user's email
- **Deep linking**: Setup page reads `?invite=TOKEN`, shows context, pre-fills fields, auto-login + accept after signup
- **Login page**: Reads `?invite=TOKEN`, accepts after login
- **Directory tab**: Invite form with email/name/message, sent invites list
- **Federation**: Cross-instance invites create federated connections

## Orchestration Layer (Kanban → Task → Route → Brief)
- **Brief Assembly Engine** (`src/lib/brief-assembly.ts`): Reads a Kanban card's full context graph (contacts, pipeline stage, checklist, relay history, activity) and assembles a structured Markdown brief
- **Skill Matching**: Cross-references required skills/taskTypes against connection profiles, scores by match quality + capacity
- **AgentBrief model** in Prisma: Stores reasoning receipts with source context, routing decisions, matched skills, result relay links
- **Action Tags**:
  - `task_route` — Decompose a card into tasks, match against connections, route via relay with brief attached
  - `assemble_brief` — Generate a context brief for any card without routing (show-your-work)
- **Briefs API**: `GET /api/briefs` (list by user, filterable by cardId/relayId), `GET /api/briefs/[id]` (single brief)
- **System Prompt Layer 17**: Updated with Kanban-Driven Orchestration rules (section 6) and Relay Preferences Awareness (section 7)

## Ambient Relay Learning System
- **Schema**: `AmbientRelaySignal` (per-relay outcome data: timing, quality, disruption, topic relevance) + `AmbientPattern` (synthesized cross-user learnings)
- **Engine** (`/src/lib/ambient-learning.ts`):
  - `captureAmbientSignal()` — Logs outcome metrics when relay_respond fires on ambient relay
  - `synthesizePatterns()` — Aggregates signals into patterns (timing, topic, frequency, latency, disruption, overall)
  - `getAmbientLearningPromptSection()` — Generates system prompt text from active patterns
  - `captureIgnoredAmbientSignals()` — Detects ambient relays delivered >48h without response, logs "ignored" signals
- **System prompt wiring**: Layer 17 calls `getAmbientLearningPromptSection()` and appends learned patterns
- **Self-assessment**: Layer 17 section 3b instructs Divi to include `_ambientQuality`, `_ambientDisruption`, `_ambientTopicRelevance`, `_conversationTopic`, `_questionPhrasing` in relay_respond tags for ambient relays
- **relay_respond tag syntax** (layer15) updated with self-assessment params
- **Background synthesis**: Chat flow triggers synthesis after any ambient signal capture (fire-and-forget)
- **API**: `POST /api/ambient-learning/synthesize` (triggers synthesis + ignored detection), `GET` (returns active patterns + stats)
- This is the feedback loop: signals → patterns → prompt → better ambient behavior → more signals

## Teams & Projects
- **Schema**: Team, TeamMember, Project, ProjectMember models in Prisma
- **Federated members**: TeamMember/ProjectMember can use `connectionId` (federated) or `userId` (local)
- **Roles**: Team: owner/admin/member | Project: lead/contributor/reviewer/observer
- **APIs**: `/api/teams`, `/api/teams/[id]`, `/api/teams/[id]/members`, `/api/projects`, `/api/projects/[id]`, `/api/projects/[id]/members`
- **UI**: TeamsView component in dashboard center panel (Teams tab) with list/detail views for both teams and projects
- **Relay routing**: `findSkillMatches` in `brief-assembly.ts` accepts optional `teamId`/`projectId` — project members get +10, team members +5 score boost
- **Action tags**: `task_route`, `assemble_brief`, `relay_broadcast`, `relay_ambient` all accept optional `teamId`/`projectId` params
- **System prompt**: Layer 17 now fetches user's teams/projects and includes them in context, plus scoped routing instructions
- **Visibility**: Project `visibility` field: `private` (only explicit members), `team` (all team members can see), `open` (any connected user can discover)
- **Project context assembler**: `assembleProjectContext` in `brief-assembly.ts` — builds cross-member dashboard (cards, queue, relays per member)
- **Project dashboard API**: `GET /api/projects/[id]/context` — returns full project context; `?format=markdown` for prompt-ready text
- **Action tag**: `project_dashboard` — Divi can invoke to see all members' activity for a project
- **Layer 17 enhancement**: Active project dashboards (up to 3) injected into system prompt, including per-member activity and blocker detection
- **Visibility enforcement**: API routes check `private`/`team`/`open` access rules; `team` visibility checks parent team membership; `open` checks any connected user
- **UI**: Projects list shows visibility badges (🔒/👥/🌐), cross-instance/cross-team indicators; project detail has "🧠 Divi's View" dashboard showing per-member activity
- **Federation docs**: `/docs/federation` has Teams & Projects section
- **KanbanCard/QueueItem/AgentRelay**: optional `projectId`; QueueItem/AgentRelay also have `teamId`

## Federation Project Context
- **Endpoint**: `GET /api/federation/project/[id]/context` — federation-facing; remote Divi can fetch cross-member project dashboards
- **Auth**: `x-federation-token` header matched against active federated Connection's `federationToken`
- **Access**: Validates connection is a ProjectMember (via `connectionId`) or connection's local user is a member
- **Reuses**: `assembleProjectContext` + `generateProjectDashboardMarkdown` from brief-assembly.ts
- **CORS**: Allows cross-origin for inter-instance calls

## Agent Extensions Framework
- **Schema**: `AgentExtension` model — name, slug, type (skill/persona/prompt_layer), source (manual/clawmart/custom), config (JSON text), scope (user/team/project/global), scopeId, priority, isActive
- **Config format**: `{ promptText?, actionTags?: [{name, description, syntax}], parameters?, model? }`
- **Loader**: Layer 19 in system-prompt.ts — fetches active extensions matching user's scope (own + teams + projects + global), injects prompt text and action tag docs
- **APIs**: `GET/POST /api/extensions`, `GET/PUT/DELETE /api/extensions/[id]` — CRUD with scope validation
- **UI**: ExtensionsView component in dashboard (🧩 Extensions tab) — list/detail/import views
- **Import flow**: Manual JSON paste with Claw Mart banner/link; supports type, source, scope, priority
- **Claw Mart**: Link to shopclawmart.com marketplace for browsing pre-built skills/personas; manual import for now (no confirmed consumer API)
