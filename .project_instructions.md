# DiviDen Command Center — Project Instructions

## Overview
DiviDen is an individual-first, opencore personal AI command center. Dark theme, command-center aesthetic. Deployed at dividen.ai and sdfgasgfdsgsdg.abacusai.app.

## Key Design Decisions
- **Publisher Attribution**: All platform items published under "DiviDen" (`publisherType: 'platform'`). User-submitted capabilities get `publisherType: 'user'` with their name.
- **Agent Skills (agentskills.io)**: Capabilities can be flagged as Agent Skills via `skillFormat`, `skillBody`, `skillSource` fields. Displayed with 🧩 badge.
- **Approval Flow**: User-submitted capabilities start as `pending_review`. Admin can approve/reject with reason. Only `approved` items visible in marketplace browse.
- **Inbox → Email**: Renamed throughout (CenterPanel, KeyboardNav, GlobalSearch).
- **Smart Task Assembly**: QueuePanel uses multi-step task creation (type → objective → context → review) instead of freeform input.
- **Network Auto-Connect**: Users on dividen.ai primary instance are auto-connected. Federation wizard hidden for primary instance.
- **Settings Loading**: All settings pages (Profile, Relay, Divi) have retry logic (2 retries, 1s delay) to handle transient DB connection pool exhaustion.

## Admin Panel
- Admin auth uses Bearer token with ADMIN_PASSWORD env var for all admin APIs
- Admin capabilities API: `/api/admin/capabilities` (CRUD with Bearer auth)
- Admin marketplace API: `/api/admin/marketplace` (expanded with POST/DELETE for agents)
- MarketplaceTab has 4 sub-tabs: Agents | Capabilities | + Capability | + Agent
- Users fetched from `/api/admin/stats` → `users.list` for publish-as-user dropdowns

## Technical Notes
- TSC can run out of memory — skip `test_nextjs_project`, go straight to `build_and_save_nextjs_project_checkpoint`
- DB is shared dev/prod — never use `--accept-data-loss`
- DATABASE_URL must be explicitly exported when running seed scripts
- Admin credentials: admin@dividen.ai / DiviDenAdmin2026!

## Visual Style
- Dark theme with glass-morphism effects
- Brand color: `brand-500` (purple-ish)
- Text: `var(--text-secondary)` for muted, white for primary
- Cards: `bg-white/[0.02]` with `border-white/[0.06]`
- Status badges: emerald (active/approved), amber (pending), red (rejected/suspended), gray (disabled)
