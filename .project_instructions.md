## Key Architecture Notes

### System Prompt (src/lib/system-prompt.ts)
- 17 prompt groups, conditionally loaded based on message relevance scoring
- `relay` group force-loaded when `inboundRelayCount > 0` OR `pendingConnectionRequests.length > 0`
- 5 CRITICAL EXECUTION RULES at ~line 975 prevent phantom work (describe-without-emit)
- Auto-continue after `sync_signal` — injected via ChatView.tsx SSE handler

### Action Tags (src/lib/action-tags.ts)
- `relay_request` (send relay), `relay_respond` (reply to inbound) — NOT `send_relay`
- `accept_connection` — accepts pending connection requests
- `task_route` — `to` field bypasses skill matching for explicit assignments
- `upsert_card` — creates/updates kanban cards

### Relay System
- `agentRelay` table: status flow is `pending` → `delivered` → `completed`/`declined`
- Inbound relays surfaced with CRITICAL instruction at START of response
- Relay responses flow through comms channel
- NOW API includes `delivered` status in relay filter

### Federation / Directory
- Instance entities without `operatorName` are hidden from directory
- Federated operators show as `source: 'federated_operator'` (not `'federated'`)
- PeerProfileModal handles federated operators with instance context card
