# DiviDen Command Center - Project Instructions

## Overview
DiviDen is an agentic working protocol command center built with Next.js 14, Prisma, and Tailwind CSS. It provides AI agent coordination, contact management, document handling, calendar integration, and inter-agent communication.

## Design System
- Dark theme with #4F7CFF brand blue, #6B9AFF brand secondary
- Space Grotesk headings, JetBrains Mono for labels/mono text
- Background: #0a0a0a base, #111111 surface
- Text: #f5f5f5 primary, #a1a1a1 secondary, #666666 muted
- Borders: rgba(255,255,255,0.06)

## Key Architecture
- Auth: NextAuth with credentials provider, admin password from ADMIN_PASSWORD env var
- Database: Prisma with PostgreSQL
- AI: Abacus.AI LLM API (ABACUSAI_API_KEY)
- Email: IMAP/SMTP integration + Abacus notification API
- Federation: Cross-instance agent communication support

## Admin Dashboard
- `/admin` route with stats, user management, system overview
- Protected by admin check (user email matches admin)

## Outbound Invitations
- **Prisma model**: `Invitation` with token (uuid), inviter/invitee info, message, status, instanceUrl, sourceInstance, expiresAt (14 days)
- **Email**: Notification type "Connection Invitation" (NOTIF_ID_CONNECTION_INVITATION), styled HTML from `invites@{hostname}`
- **API routes**:
  - `POST /api/invites` — create + send email (or auto-create connection if user exists)
  - `GET /api/invites` — list sent invites
  - `GET /api/invites/[token]` — public peek for signup context
  - `POST /api/invites/[token]/accept` — accept specific invite
  - `POST /api/invites/accept-pending` — auto-accept all pending invites for user's email
- **Deep linking**: Setup page reads `?invite=TOKEN`, shows context, pre-fills fields, auto-login + accept after signup
- **Login page**: Reads `?invite=TOKEN`, accepts after login
- **Directory tab**: Invite form with email/name/message, sent invites list
- **Federation**: Cross-instance invites create federated connections
