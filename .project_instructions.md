# DiviDen Command Center

## Overview
DiviDen Command Center is a personal coordination hub between humans and AI agents. It features a dark-themed dashboard with three panels (NOW, Center, Queue), AI chat with streaming, Kanban board with drag-and-drop, CRM, 3-tier memory system, webhook infrastructure, and Agent API v2.

## Technical Stack
- Next.js 14 (App Router) with TypeScript
- PostgreSQL via Prisma ORM
- NextAuth.js v4 (credentials provider, JWT sessions)
- Abacus AI RouteLLM API for AI chat (model: gpt-4.1)
- @dnd-kit for Kanban drag-and-drop
- Tailwind CSS with custom dark theme

## Database Models (added)
- **Document**: title, content (markdown), type (note/report/template/meeting_notes), tags, optional card link
- **Recording**: title, source, transcript, summary, duration, status (pending/processed/reviewed), optional card link
- **ActivityLog**: action, actor (user/divi/system), summary, metadata, userId
- **CommsMessage**: sender (user/divi/system), content, state (new/read/acknowledged/resolved/dismissed), priority (urgent/normal/low), linkedCardId, linkedContactId, linkedRecordingId, linkedDocumentId, metadata, userId

## Comms Channel
- Dedicated page at /dashboard/comms (NOT a 4th dashboard panel â€” dashboard stays 3-panel)
- Accessed via envelope icon button in the dashboard top header bar (with unread badge)
- Also accessible from mobile bottom nav bar
- Bidirectional task passing between operator and Divi with state lifecycle tracking
- Messages auto-mark as "read" when selected
- Action buttons: Mark Read, Acknowledge, Resolve, Dismiss, Delete, Reply
- Compose UI with priority selector (urgent/normal/low)
- Messages can link to KanbanCard, Contact, Recording, Document
- API routes: GET/POST /api/comms, PATCH/DELETE /api/comms/[id], GET /api/comms/unread

## Auth Flow
- First-run: /setup creates initial admin account
- Login: /login with credentials
- Protected: /dashboard and /settings require authentication
- SessionProvider wraps entire app

## LLM Integration
- Uses Abacus AI API (ABACUSAI_API_KEY env var)
- Streaming via fetch to https://apps.abacus.ai/v1/chat/completions
- Model: gpt-4.1 with 4096 max tokens
- The original app also supports user-provided OpenAI/Anthropic keys stored in DB (AgentApiKey model) but the primary LLM is via Abacus AI
