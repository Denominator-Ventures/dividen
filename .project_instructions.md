# DiviDen Command Center ŌĆö Project Instructions

## Overview
DiviDen is an AI-powered command center / CRM / productivity platform. Users interact with their AI agent "Divi" (customizable name) through chat. The platform includes Kanban board, CRM, calendar, email integration, marketplace, and a relay/federation network.

## Key Architecture
- **Next.js 14** App Router + Prisma ORM + PostgreSQL (shared dev/prod DB)
- **System Prompt**: `src/lib/system-prompt.ts` ŌĆö Context Relevance Engine selects from 13 prompt groups based on message content
- **Action Tags**: `src/lib/action-tags.ts` ŌĆö `[[tag_name:{params}]]` pattern parsed from LLM responses and executed server-side
- **Auth**: NextAuth with credentials + Google OAuth for integrations (not SSO)
- **LLM**: Multi-provider via `src/lib/llm.ts` (OpenAI, Anthropic, Gemini, etc.)

## Onboarding System (Phased, Chat-Based)
- **Schema**: `User.onboardingPhase` (Int, 0-6) tracks progress
- **Phases**: 0=not started, 1=personalization (working style, triage, goals, name), 2=Google connect, 3=platform tour, 4=webhooks, 5=launch/propagation, 6=complete
- **API**: `/api/onboarding/advance` (POST) ŌĆö handles `init`, `advance`, `skip`, `show_settings`
- **Widgets**: `OnboardingChatWidgets.tsx` ŌĆö renders slider, toggle, radio, select, text_input, google_connect, info, webhook_setup, submit, skip
- **Settings Re-invocation**: `[[show_settings_widget:{"group":"working_style|triage|goals|identity|all"}]]` action tag in `action-tags.ts`
- **System Prompt Awareness**: `buildSetupLayer_conditional` includes onboarding phase status + settings widget hint; `setup` group is force-included during active onboarding
- **Phase 5 Propagation**: Triggers catch-up message when user clicks "Start Propagation"
- **Phase -1**: Settings adjustment (non-onboarding) ŌĆö reuses widget renderer with `onboardingPhase: -1`

## Visual Style
- Dark theme with CSS variables (`--bg-primary`, `--bg-surface`, `--brand-primary`, etc.)
- Glassmorphism elements with `bg-white/[0.03]` and `border-white/[0.06]`
- Brand color: `var(--brand-primary)` (purple/blue tones)
- Text hierarchy: `--text-primary`, `--text-secondary`, `--text-muted`

## Feedback System
- **Schema**: `Feedback` model (id, userId, category, message, rating, page, status, adminNote)
- **API**: `/api/feedback` ŌĆö GET (admin list), POST (user submit), PATCH (admin update status/note)
- **User-facing**: `FeedbackTab` floating tab on right edge of dashboard
- **Admin-facing**: `FeedbackAdminTab` in admin page ŌĆö filter by status, expand items, set status, add admin notes
- **Categories**: general, bug, feature, onboarding, ux
- **Statuses**: new, reviewed, resolved, archived

## Important Notes
- DB is shared dev/prod ŌĆö never use `--accept-data-loss`
- Use `NODE_OPTIONS="--max-old-space-size=10240"` for builds (TSC OOM risk)
- Package manager: yarn only
- Deployed at: `dividen.ai` and `sdfgasgfdsgsdg.abacusai.app` (both untagged)
- Prisma `workingStyle`/`triageSettings` are `JsonValue` ŌĆö use `String()` wrapper before `JSON.parse()`
