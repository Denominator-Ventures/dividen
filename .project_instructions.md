## Queue Confirmation Gate & CoS Execution Engine
- **Confirmation gate**: All queue items enter as `pending_confirmation` by default. User must approve before Divi acts.
- **queueAutoApprove**: User-level boolean in User model (default false). When true, items skip confirmation and go straight to `ready`.
- **Action tags**: `dispatch_queue` and `queue_capability_action` in `src/lib/action-tags.ts` check `queueAutoApprove` to set initial status.
- **Confirm endpoint**: `POST /api/queue/confirm` (session-based) and `POST /api/v2/queue/{id}/confirm` (Bearer token) — accepts `{ queueId, action: 'approve'|'reject' }`.
- **QueuePanel UI**: `src/components/dashboard/QueuePanel.tsx` renders a `pending_confirmation` section with ✅/❌ buttons.
- **CoS mode**: `src/lib/cos-sequential-dispatch.ts` — proactive execution engine. `executeTask()` parses handler to determine strategy (capability/relay/generic), performs the action, logs via `logActivity()`, advances to next task.
- **logActivity()** takes an object: `logActivity({ userId, action, actor, summary })`.
- **v2 Settings API**: `GET/PATCH /api/v2/settings` — reads/writes `mode` and `queueAutoApprove`.
- **OpenAPI docs**: Updated with `pending_confirmation` status, `/queue/{id}/confirm`, `/settings` endpoints.
- **Developer docs**: 3 new sections — Queue Gating & Confirmation, CoS Execution Engine, Settings API v2.
- **Release update**: Entry added to `src/lib/updates.ts` covering all changes.
