# DiviDen Command Center

## Overview
DiviDen Command Center is a personal coordination hub between humans and AI agents. It features a dark-themed dashboard with three panels (NOW, Center, Queue), AI chat with streaming, Kanban board with drag-and-drop, CRM, 3-tier memory system, webhook infrastructure, and Agent API v2.

## Technical Stack
- Next.js 14 (App Router) with TypeScript
- PostgreSQL via Prisma ORM
- NextAuth.js v4 (credentials provider, JWT sessions)
- Abacus AI RouteLLM API for AI chat (model: gpt-4.1)
- @dnd-kit for Kanban drag-and-drop
- Tailwind CSS with custom dark theme

## Project Structure
- Source code in `src/` directory (`@/*` → `./src/*`)
- `src/app/` - App Router pages and API routes
- `src/components/` - Dashboard and settings components
- `src/lib/` - Core utilities (auth, prisma, llm, memory, action-tags, etc.)

## Key Features
- **Dashboard**: 3-panel layout (NOW/Center/Queue) with mobile single-panel view + bottom nav
- **Agent Name**: "Divi" — the AI agent identity (open source default name)
- **Chat**: SSE streaming with 13-layer system prompt and action tag parsing, placeholder "Message Divi..."
- **Kanban**: Drag-and-drop board with 10 pipeline stages (leads→qualifying→proposal→negotiation→contracted→active→development→planning→paused→completed)
- **CRM**: Contact management with enrichment, activity timeline, relationship mapping
- **ContactRelationship**: Model linking contacts with 8 types (colleague/manager/report/partner/spouse/friend/referral/custom)
- **ContactDetailModal**: 3-tab layout (Overview | Activity | Relationships) with stats row (cards/emails/events/comms/relations)
- **Activity API**: GET /api/contacts/[id]/activity returns unified timeline + relationships + stats
- **Relationships API**: GET/POST/DELETE /api/contacts/[id]/relationships for CRUD
- **Recordings**: Generic recording/transcript tab — supports Plaud, Otter, Fireflies, or any note-taker via webhooks
- **Drive/Documents**: Create, edit, delete documents (notes, reports, templates, meeting notes) with search
- **Activity Log**: System-wide activity feed tracking card moves, dispatches, mode changes, etc.
- **NowPanel**: Today's Pulse stats (pipeline/divi tasks/active/blocked), Portfolio section (active companies), quick stats row
- **QueuePanel**: "Divi's Queue" + "Activity" sub-tabs
- **Memory**: 3-tier system (Facts/Rules/Patterns)
- **Queue**: Task management with dispatch logic
- **Webhooks**: Calendar, email, transcript, generic webhook types
- **Agent API v2**: External agent authentication with Bearer tokens

## Connections & Agent Relay (Phase 1)
- **Connection model**: Links two users (local or federated) with status lifecycle (pending/active/blocked/declined)
- **Permissions**: JSON field with trustLevel (full_auto/supervised/restricted) and scopes array (request_files/assign_tasks/read_status/schedule/share_updates/request_approval)
- **AgentRelay model**: Structured request/response protocol between connected agents with status lifecycle (pending/delivered/agent_handling/user_review/completed/declined/expired)
- **Relay types**: request, response, notification, update; Intents: get_info, assign_task, request_approval, share_update, schedule, introduce, custom
- **Federation**: Cross-instance connections via InstanceRegistry + FederationConfig
  - FederationConfig: mode (closed/allowlist/open), allowInbound, allowOutbound, requireApproval
  - InstanceRegistry: known remote instances with API keys and trust levels
  - Federation endpoints: POST /api/federation/connect (receive inbound connection), POST /api/federation/relay (receive inbound relay)
  - Uses X-Federation-Token header for authentication
- **ConnectionsView**: Center Panel tab with sub-tabs (Connections | Relays), connection CRUD, relay management
- **Settings**: Federation tab with mode selector, controls, known instances management
- **API routes**: /api/connections, /api/connections/[id], /api/relays, /api/relays/[id], /api/relays/counts
- **Action tags (4 new, 26 total)**: relay_request, accept_connection, relay_respond, update_profile
- **System prompt Layer 17**: Connections & Agent Relay awareness — feeds active connections + pending relays + behavioral rules
- **System prompt Layer 18**: Profile Awareness — feeds user's own profile + connected users' profiles for relay routing intelligence
- **Walkthrough**: New "Connections & Agent Relay" step added

## Personal Profiles (Lived Experience)
- **UserProfile model**: `user_profiles` table — professional (headline, bio, skills, experience, education), lived experience (languages, countriesLived, lifeExperiences, volunteering, hobbies, personalValues, superpowers), availability (capacity, capacityNote, timezone, workingHours, outOfOffice), privacy (visibility, sharedSections)
- **Field mapping**: DB `capacity` = UI `capacityStatus`; DB `lifeExperiences` = UI `lifeMilestones`; no `certifications` in DB
- **Create pattern**: use `user: { connect: { id: userId } }` not raw `userId`
- **API routes**: `/api/profile` (GET/PUT own), `/api/profile/[userId]` (GET peer w/ privacy), `/api/profile/import-linkedin` (POST w/ LLM)
- **UI**: Settings → 👤 Profile tab (ProfileEditor component, 5 sub-tabs)
- **ConnectionsView**: "View Profile" peek on expanded connections
- **Action tag `update_profile`**: Merges arrays (doesn't replace), supports field remapping
- **Layer 18 routing rules**: Skills + lived experience + availability; lived-in > speaks-language for cultural understanding; capacity "busy/unavailable" = route elsewhere; superpowers = unique match priority

## Webhook Auto-Learn + Manual Mapping
- When a webhook payload arrives and no mapping exists, the system triggers LLM analysis in the background
- LLM analyzes payload structure and produces a field map (dot-notation paths → standard field names)
- Field maps stored in Webhook.mappingRules as JSON (`WebhookMappingConfig` format)
- Users can view/edit/clear mappings in Settings > Webhook Manager > "🧠 Field Mapping" button
- "Ask Divi" button triggers manual re-learn via LLM
- Per-type field templates: calendar (title/description/startTime/endTime/location/attendees), email (subject/fromName/fromEmail/toEmail/body/snippet), transcript (title/transcript/speakers/duration), generic (title/content/source)
- Core module: `src/lib/webhook-learn.ts` (analyzePayload, learnAndSaveMapping, parseMappingConfig)
- API: GET/POST `/api/webhooks-management/[id]/learn`
- Webhook routes extract with fieldMap first, fall back to sniffing
- Uses user's own LLM keys (OpenAI/Anthropic from AgentApiKey table)

## Platform Setup Assistant (Layer 16)
- Divi can configure the platform directly from chat when user asks for help
- Two modes: "Do It For Them" (action tags) or "Guide Them" (step-by-step instructions)
- Layer 16 dynamically loads current platform state (active API keys, webhooks, counts)
- New action tags: setup_webhook, save_api_key, create_calendar_event, create_document, send_comms
- Divi knows how to guide external integrations: Google Calendar, Gmail, Outlook, Plaud, Otter, Fireflies, Zapier/Make/n8n
- Behavioral: if user pastes API key → auto-save; if user says "set up" → offer to create webhook; proactively suggests missing setup

## Global Search
- Command palette UI triggered by ⌘K / Ctrl+K or search button in header
- Searches across: KanbanCard, Contact, Document, Recording, CalendarEvent, EmailMessage, CommsMessage, QueueItem
- API: GET /api/search?q=query&limit=25 (case-insensitive, 250ms debounce)
- Results grouped by type with color-coded badges, keyboard navigation (↑↓ Enter Esc)
- Selecting a result navigates to the relevant Center Panel tab (or /dashboard/comms for comms)
- Component: GlobalSearch.tsx

## Auth Flow
- First-run: /setup creates initial admin account

- Protected: /dashboard and /settings require authentication
- SessionProvider wraps entire app

## LLM Integration
- Uses Abacus AI API (ABACUSAI_API_KEY env var)
- Streaming via fetch to https://apps.abacus.ai/v1/chat/completions
- Model: gpt-4.1 with 4096 max tokens
- The original app also supports user-provided OpenAI/Anthropic keys stored in DB (AgentApiKey model) but the primary LLM is via Abacus AI
