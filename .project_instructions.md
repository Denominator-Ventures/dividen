## Queue Confirmation Gate & CoS Execution Engine
- **Confirmation gate**: All queue items enter as `pending_confirmation` by default. User must approve before Divi acts.
- **queueAutoApprove**: User-level boolean in User model (default false). When true, items skip confirmation and go straight to `ready`.
- **Action tags**: `dispatch_queue` and `queue_capability_action` in `src/lib/action-tags.ts` check `queueAutoApprove` to set initial status.
- **Confirm endpoint**: `POST /api/queue/confirm` (session-based) and `POST /api/v2/queue/{id}/confirm` (Bearer token) — accepts `{ queueId, action: 'approve'|'reject' }`.
- **Chat-based queue control**: 3 new action tags — `confirm_queue_item` (approve pending→ready), `remove_queue_item` (delete), `edit_queue_item` (update + trigger smart prompter). System prompt instructs Divi to offer confirm/edit/remove after proposing items.
- **Smart Task Prompter v2**: `src/lib/smart-task-prompter.ts` — Agent-aware optimization engine. On edit, it:
  1. Resolves the target agent's Integration Kit from MarketplaceAgent (taskTypes, requiredInputSchema, contextInstructions, usageExamples, executionNotes).
  2. Calls LLM to produce `displaySummary` (≤120 chars for queue card UI) and `optimizedPayload` (full structured payload matching the agent's input schema).
  3. Stores both in `metadata` JSON (never overwrites title/description). Also stores `_original`, `_optimizedAt`, `_optimizedForAgent`.
  4. Falls back to generic `{ task, context, deliverables, files, constraints }` structure when no agent schema is available.
- **Inline edit UI**: QueueItemCard has ✏️ button → inline form (title, desc, priority) → "Save & Optimize" triggers PATCH + smart prompter. Shows "✨ optimizing..." pulse while running. ⚡ badge shown when optimizedPayload exists.
- **Queue card display**: Shows `displaySummary` as headline (if available), full title below if truncated, description up to 2 lines. Full title on hover.
- **CoS relay dispatch**: `cos-sequential-dispatch.ts` sends `optimizedPayload` in relay when available, falls back to raw description.
- **QueuePanel UI**: `src/components/dashboard/QueuePanel.tsx` renders a `pending_confirmation` section with ✅/❌ buttons.
- **CoS mode**: `src/lib/cos-sequential-dispatch.ts` — proactive execution engine. `executeTask()` parses handler to determine strategy (capability/relay/generic), performs the action, logs via `logActivity()`, advances to next task.
- **logActivity()** takes an object: `logActivity({ userId, action, actor, summary })`.
- **v2 Settings API**: `GET/PATCH /api/v2/settings` — reads/writes `mode` and `queueAutoApprove`.
- **OpenAPI docs**: Updated with `pending_confirmation` status, `/queue/{id}/confirm`, `/settings` endpoints.
- **Developer docs**: 3 new sections — Queue Gating & Confirmation, CoS Execution Engine, Settings API v2.
- **Release update**: Entry added to `src/lib/updates.ts` covering all changes.
