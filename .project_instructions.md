## Federation Push & Ack Loop (v2.1.3+)
- `src/lib/federation-push.ts` — three exported functions:
  - `pushRelayToFederatedInstance()` — POSTs to remote `/api/federation/relay`, on 200 ack: logs activity, stores `peerRelayId`, posts comms confirmation
  - `pushRelayAckToFederatedInstance()` — POSTs completion/decline back to originating instance's `/api/federation/relay-ack`
  - `pushNotificationToFederatedInstance()` — lightweight notification push (project invites, etc.)
- `/api/federation/relay-ack` (NEW) — inbound completion callback from remote instance. On receipt:
  1. Updates relay status → completed/declined
  2. Logs activity (`federation_relay_completed`)
  3. Posts comms message to sender
  4. Advances queue item → `done_today`, triggers sequential dispatch
  5. Updates checklist item delegation status
  6. Checks card auto-completion
  7. Fires webhook `relay_state_changed`
- `relay_respond` action tag: now pushes ack back to originating instance if relay has `peerRelayId`
- `PATCH /api/relays/[id]`: same ack-back wiring for UI-driven completions
- Sending Divi owns the loop — every push expects an ack, every completion flows back
